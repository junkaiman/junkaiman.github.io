You could add

location /jupyterhub {
    proxy_pass http://localhost:8000;
}
to your nginx configuration. This forwards all trafic on the /jupyterhub path to the proxy.

总结

随便修理修理…也是个烂摊子………

搞定了同一个目录下的同步协作，在conf里修改spawn路径就行，但是伴随而来的是很多用户权限的管理…..

多用户在一个目录下不同文件同步还行，如果是同步编辑同一个文件就不太可了….

要关闭再打开才能看到修改，而且不chmod 777的话，大家权限都不一样，读写起来不一致….
